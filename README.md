# Executive Project Dashboard - โรงเรียนบ้านห้วยแก้วสามัคคีธรรม

ระบบเว็บแดชบอร์ดติดตามแผนงานโครงการสำหรับผู้บริหาร โรงเรียนบ้านห้วยแก้วสามัคคีธรรม แสดงผลสรุปสถิติ งบประมาณจัดสรร งบประมาณที่ใช้ไป งบประมาณคงเหลือ และความก้าวหน้าโครงการแบบเรียลไทม์ (Real-time) โดยดึงข้อมูลตรงจาก Google Sheets

## ฟีเจอร์หลัก (Key Features)
1. **การเชื่อมโยงข้อมูลแบบเรียลไทม์:** เชื่อมต่อกับไฟล์ Google Sheets โดยตรง ทุกครั้งที่โหลดหน้าเว็บระบบจะประมวลผลข้อมูลล่าสุดเสมอ
2. **สรุปตัวชี้วัดหลัก (KPI Cards):** แสดงจำนวนโครงการทั้งหมด งบประมาณจัดสรรรวม งบประมาณที่เบิกจ่ายแล้ว งบประมาณคงเหลือ และความก้าวหน้าเฉลี่ยพร้อมรายละเอียดสถิติย่อย
3. **การนำเสนอด้วยแผนภูมิเชิงตอบสนอง (Interactive Charts):**
   - แผนภูมิวงกลมโดนัท (Doughnut Chart) แสดงสัดส่วนสถานะโครงการ
   - แผนภูมิแท่งเปรียบเทียบ (Bar Chart) งบประมาณจัดสรรและงบประมาณที่ใช้จริงแยกตามกลุ่มงาน
4. **เครื่องมือค้นหาและคัดกรองข้อมูล:** ค้นหาโครงการด่วน กรองแยกตามกลุ่มงาน หรือกรองแยกตามสถานะการดำเนินงาน
5. **การจัดเรียงข้อมูลแบบไดนามิก:** สามารถคลิกที่หัวตาราง (Table Header) เพื่อจัดเรียงข้อมูลตามคอลัมน์จากน้อยไปมาก หรือมากไปน้อยได้
6. **โหมดการแสดงผลแบบพรีเมียม:** รองรับโหมดมืด (Dark Mode) และโหมดสว่าง (Light Mode) ที่ปรับเปลี่ยนโทนสีและแผนภูมิให้สวยงามโดยอัตโนมัติ

## ขั้นตอนการอัปโหลดขึ้น GitHub และเปิดใช้งาน GitHub Pages

เพื่อแชร์แดชบอร์ดนี้ให้คณะผู้บริหารเข้าชมผ่านอินเทอร์เน็ต สามารถนำโค้ดขึ้น GitHub ได้ 2 ช่องทางดังนี้:

### ช่องทางที่ 1: การอัปโหลดผ่านหน้าเว็บไซต์ GitHub (แนะนำ - สะดวกที่สุดโดยไม่ต้องใช้คำสั่ง)

1. เปิดเว็บไซต์ [GitHub](https://github.com) และลงชื่อเข้าใช้งานบัญชีของคุณ
2. คลิกปุ่ม **New** (หรือเครื่องหมายบวก `+` ที่มุมขวาบน เลือก *New repository*)
3. ตั้งชื่อ Repository เช่น `huaykaew-project-dashboard`
4. เลือกการเข้าถึงเป็น **Public** (สาธารณะ เพื่อให้ใช้งาน GitHub Pages ได้)
5. ปล่อยตัวเลือกอื่นๆ ไว้ตามเดิม แล้วคลิกปุ่ม **Create repository**
6. ในหน้าถัดไป ให้มองหาข้อความ *"Get started by creating a new file or uploading an existing file."* แล้วคลิกที่ลิงก์ **upload an existing file**
7. เปิดโฟลเดอร์ของโครงการในคอมพิวเตอร์ของคุณ จากนั้นลากและวางไฟล์ทั้งหมด (อย่างน้อย 4 ไฟล์ได้แก่ `index.html`, `styles.css`, `app.js` และ `README.md`) ลงในพื้นที่ลากวางบนหน้าเว็บ GitHub
8. รอให้ไฟล์อัปโหลดเสร็จเรียบร้อย จากนั้นเลื่อนลงมาล่างสุดแล้วคลิกปุ่ม **Commit changes** (สีเขียว)
9. ดำเนินงานต่อที่หัวข้อ **ขั้นตอนการเปิดใช้งานหน้าเว็บไซต์ผ่าน GitHub Pages** (ด้านล่าง) ได้ทันที

### ช่องทางที่ 2: การอัปโหลดผ่านชุดคำสั่ง Git (สำหรับผู้ที่มีโปรแกรม Git ในเครื่องแล้ว)

### ขั้นตอนที่ 1: เตรียมโค้ดในเครื่องคอมพิวเตอร์ของคุณ
หากยังไม่ได้สร้าง Git Repository ในโฟลเดอร์นี้ ให้เปิด Command Prompt / PowerShell หรือ Terminal ในโฟลเดอร์โครงการ (`c:\Users\Advice SB\Desktop\kookkik`) แล้วรันคำสั่งต่อไปนี้:

```bash
# 1. เริ่มต้นระบบ Git ในโฟลเดอร์
git init

# 2. เพิ่มไฟล์ทั้งหมดเข้าในระบบคิวเตรียมบันทึก
git add .

# 3. บันทึกประวัติครั้งแรก
git commit -m "Initial commit: ระบบ Dashboard ติดตามแผนงานโครงการ"
```

### ขั้นตอนที่ 2: สร้าง Repository บน GitHub
1. เปิดเว็บไซต์ [GitHub](https://github.com) และลงชื่อเข้าใช้งานบัญชีของคุณ
2. คลิกปุ่ม **New** (หรือเครื่องหมายบวก `+` ที่มุมขวาบน เลือก *New repository*)
3. ตั้งชื่อ Repository เช่น `huaykaew-project-dashboard`
4. เลือกการเข้าถึงเป็น **Public** (สาธารณะ เพื่อให้สามารถใช้งาน GitHub Pages เผยแพร่เว็บได้)
5. **ห้าม** ติ๊กเลือก *Add a README file*, *Add .gitignore* หรือ *Choose a license* เนื่องจากเรามีไฟล์เหล่านี้อยู่แล้ว
6. คลิกปุ่ม **Create repository**

### ขั้นตอนที่ 3: เชื่อมโยงโฟลเดอร์ของคุณกับ GitHub และผลักข้อมูล (Push)
หลังจากสร้าง Repository แล้ว GitHub จะแสดงหน้ารายละเอียดชุดคำสั่ง ให้นำคำสั่งในหัวข้อ *"…or push an existing repository from the command line"* มารันในคอมพิวเตอร์ของคุณ ดังนี้:

```bash
# 1. สร้างกิ่งหลักชื่อ main
git branch -M main

# 2. เชื่อมโยงโฟลเดอร์ในเครื่องกับ GitHub (แทนที่ USERNAME และ REPO ด้วยข้อมูลของคุณ)
git remote add origin https://github.com/USERNAME/huaykaew-project-dashboard.git

# 3. ผลักไฟล์ในเครื่องขึ้นระบบ GitHub
git push -u origin main
```
*(หมายเหตุ: ระบบอาจขอให้คุณเข้าสู่ระบบ GitHub หรือป้อน Personal Access Token เพื่อยืนยันตัวตน)*

### ขั้นตอนที่ 4: เปิดใช้งานหน้าเว็บไซต์ผ่าน GitHub Pages
เมื่ออัปโหลดไฟล์ขึ้น GitHub เรียบร้อยแล้ว ให้ทำตามขั้นตอนการเปิดใช้บริการโฮสติ้งฟรีของ GitHub Pages:
1. เข้าไปที่หน้า Repository ของคุณบนเว็บไซต์ GitHub
2. คลิกเมนู **Settings** (รูปฟันเฟืองที่แถบเมนูด้านบน)
3. ที่เมนูด้านซ้าย เลือกหัวข้อ **Pages** (ภายใต้หมวดหมู่ *Code and automation*)
4. ในส่วนของ **Build and deployment** -> **Source** ให้เลือกเป็น **Deploy from a branch**
5. ในส่วนของ **Branch** ปรับเปลี่ยนดังนี้:
   - เปลี่ยนจาก `None` เป็น **`main`**
   - ถัดไปในส่วนโฟลเดอร์ ปล่อยไว้เป็น **`/ (root)`**
6. คลิกปุ่ม **Save**
7. รอประมาณ 1-2 นาที จากนั้นให้รีเฟรชหน้า Settings -> Pages คุณจะเห็นลิงก์หน้าเว็บไซต์ปรากฏขึ้นที่แถบสีเขียวด้านบน เช่น:
   `https://USERNAME.github.io/huaykaew-project-dashboard/`

สามารถคัดลอกลิงก์ดังกล่าวส่งให้ผู้บริหารหรือผู้เกี่ยวข้องเปิดใช้งานได้ทันทีจากทุกอุปกรณ์!
